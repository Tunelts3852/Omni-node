# 기능군별 백로그 (P0~P7)

업데이트 기준: 2026-03-13

## 1. 운영 규칙
- 활성 단계에서 구현 가능한 최소 단위 1개를 수행한다.
- 단계 완료 조건 충족 시 자동 승급한다.

## 2. 단계별 백로그

| 단계 | 기능군 | 작업 항목 | 완료 조건 |
|---|---|---|---|
| P0 | 기반 정책 | 키 소스 정책 타입/실행 정책 고정 | 문서/스크립트 정책 일치 |
| P1 | 검색 계층 | SearchGateway, GeminiGroundedRetriever | 기본 검색 성공 |
| P2 | 근거 계층 | Evidence Pack Builder, 인용 구조 | 채널 주입 가능 |
| P3 | 검증 계층 | Answer Guard, fail-closed, count-lock | 부정확 출력 차단 |
| P4 | 생성 계층 | ProviderAgnosticGeneratorAdapter | 4개 생성기 공통 동작 |
| P5 | 채널 통합 | 대화/코딩/텔레그램 공통 경로 | 경로별 회귀 통과 |
| P6 | 운영 관측 | 로그/메트릭/릴리스 게이트 | 운영 점검 가능 |
| P7 | 안정화 | 성능/보안/복구/릴리스 정리 | 배포 가능 상태 |

## 3. 핵심 우선순위
1. 키 정책 강제
2. 검색-근거-검증 파이프라인
3. 채널 공통 적용
4. 운영 관측
