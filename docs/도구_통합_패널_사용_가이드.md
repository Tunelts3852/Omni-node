# 도구 통합 패널 사용 가이드

업데이트 기준: 2026-03-13

## 1. 개요
도구 통합 패널은 설정 탭에서 `provider/tool/rag` 실행 상태를 한 화면에서 관측하고 제어하는 운영 패널입니다.

- `provider`: Groq, Gemini, Cerebras, Copilot 실행 상태와 실패 사유 확인
- `tool`: sessions/cron/browser/canvas/nodes/telegram(stub) 제어 결과 확인
- `rag`: web/memory 도구 실행 결과 확인

실무 메모:

- 이 패널은 설정 탭의 `작업 계획`, `환경 진단`, `프로젝트 문맥`과 별개로 운영/관측용이다.
- 대시보드 `로직` 탭은 노드 캔버스 편집기이고, 이 패널은 provider/tool/rag 실행 상태를 보는 관측 패널이다.
- 즉, “무엇을 할지”는 계획 패널, “지금 상태가 어떤지”는 이 패널로 보면 된다.

## 2. 화면 구성

### 2.1 상단 요약 카드
- `provider`, `tool`, `rag` 카드로 최근 실행 수, 오류 수, 최신 상태를 즉시 확인
- 카드를 클릭하면 하단 표/로그가 해당 도메인 중심으로 필터링

### 2.2 Provider 상태 표
- 제공자별 상태(`ready`, `api_key_missing`, `error`)와 실행 성공/실패/진행 건수 확인
- 라우팅 오류나 키 누락 시 근거(reason) 텍스트로 원인 확인

### 2.3 제어 입력 영역
다음 요청을 설정 탭에서 즉시 실행할 수 있습니다.

- `sessions`: list/history/send/spawn
- `cron`: status/create/toggle/delete
- `browser`: status/open/close
- `canvas`: status/present
- `nodes`: status/pending/invoke
- `telegram(stub)`: 명령 시뮬레이션
- `web`: search/fetch
- `memory`: search/get

## 3. Groq 모델 / 제한량 / 사용량 표

## 3.1 컬럼 의미
- `RPM`: 분당 요청 한도
- `RPD`: 일당 요청 한도
- `TPM`: 분당 토큰 한도
- `TPD`: 일당 토큰 한도
- `ASH`: 시간당 오디오 초 한도
- `ASD`: 일당 오디오 초 한도
- `사용량(분/시/일)`: 현재 분/시/일 윈도우 기준 누적 사용량(자동 리셋)
- `라이브 잔여/리셋`: 응답 헤더 기준 남은 요청/토큰과 리셋 힌트

## 3.2 자동 리셋/새로고침 동작
- 사용량 집계는 분/시/일 경계에서 자동으로 기준점이 갱신됩니다.
- 인증 상태(`세션 인증됨`)에서는 매 분 자동으로 Groq 모델 정보가 재조회됩니다.
- 필요 시 `새로고침` 버튼으로 즉시 재조회 가능합니다.

## 4. 모델 적용 버튼
Groq 영역의 `모델 적용` 버튼은 **전역 기본 Groq 모델 선택값**을 변경합니다.

- 적용 대상: 단일/오케스트레이션/다중 모드에서 `Groq`를 선택했을 때의 기본 모델
- 영향: 이후 전송 요청에서 별도 모델을 지정하지 않으면 이 모델이 기본으로 사용됨
- 주의: 이미 실행 중인 요청의 모델은 변경되지 않음(다음 요청부터 반영)

## 5. 운영 권장 순서
1. Provider 카드에서 `api_key_missing/error` 먼저 해소
2. Groq/Copilot 모델 표에서 기본 모델 확정
3. Tool/RAG 제어 요청으로 개별 기능 점검
4. 하단 결과 테이블의 오류 행을 우선 해결
