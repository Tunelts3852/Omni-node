# Omni-node 토큰 설정 및 메모리 초기화 가이드

업데이트 기준: 2026-03-10

## 1) 현재 토큰 설정값 (코드 기준)

### 대화탭
- 기본 최대 출력 토큰: `8192`
- 환경변수: `OMNINODE_CHAT_MAX_OUTPUT_TOKENS`

### 코딩탭
- 기본 최대 출력 토큰: `16384`
- 환경변수: `OMNINODE_CODING_MAX_OUTPUT_TOKENS`

### 텔레그램 봇
- Fast 모드 최대 출력 토큰: `1024`
- Complex 모드 최대 출력 토큰: `1536`
- 메시지 최대 길이(문자): `2000`

참고:

- 텔레그램 일반 대화는 이제 일반 질문에서 불필요한 자연어 명령 해석 LLM 1회를 건너뛰고 바로 대화 경로로 들어갑니다.

## 2) 메모리 구조/분리 정책 (최신)

- 공통 상태 루트: `~/.omninode`
- 대화 스토어: `~/.omninode/conversations.json` (최근 코딩 결과 snapshot 포함)
- 메모리 노트: `~/.omninode/memory-notes`
- 코딩 실행 이력: `~/.omninode/code-runs`
- 코딩 생성 파일: `workspace/coding/runs`

현재 정책:
- 대화탭(`chat:*`)과 코딩탭(`coding:*`)은 스코프 기준으로 분리 관리된다.
- 세션 준비 시 스코프/모드 불일치 `conversationId`는 재사용하지 않고 새 스레드를 생성한다.
- 강제 메모리/RAG 컨텍스트 단계에서 스코프에 맞는 메모리 검색 결과만 반영한다.
- 텔레그램은 `chat:single` 공유 스레드(`telegram-link` 태그)와 연동되어 대화탭과 문맥 공유가 가능하다.

## 3) 구현된 메모리 초기화 기능

## 3-1) 웹(대화탭/코딩탭)

- 대화 목록 패널에 `메모리 초기화` 버튼 제공
- 실행 시 WebSocket `clear_memory` 호출
- 스코프별 삭제:
  - `scope=chat`: 대화탭 메모리 초기화
  - `scope=coding`: 코딩탭 메모리 초기화
  - `scope=all`: 전체 초기화 (내부 API 지원)

삭제 대상:
- 해당 스코프 대화 이력(Conversation)
- 해당 스코프 메모리 노트(Memory Note)
- 해당 대화에 붙어 있던 최근 코딩 결과 snapshot
- 최근 코딩 결과의 파일 프리뷰/실행 메타데이터

## 3-2) 텔레그램 봇

- 명령: `/memory clear`
- 자연어: `메모리 초기화해줘`, `메모리 비워줘`
- 내부적으로 `scope=telegram`으로 처리되며 `chat` 스코프 메모리를 초기화한다.
- 즉, 텔레그램 연동 대화와 대화탭 공유 메모리(채팅 영역)가 함께 초기화된다.

## 3-3) 기존 단건 관리 기능

- 개별 대화 삭제: `delete_conversation`
- 메모리 노트 조회:
  - `list_memory_notes`
  - `read_memory_note`

## 4) 수동 전체 초기화 방법 (강제 리셋)

```bash
# 1) 서버 중지
pkill -f OmniNode.Middleware || true

# 2) 대화 이력 전체 삭제
rm -f ~/.omninode/conversations.json

# 3) 메모리 노트 전체 삭제
rm -rf ~/.omninode/memory-notes/*

# 4) (선택) 코딩 실행 이력도 삭제
rm -rf ~/.omninode/code-runs/*

# 5) (선택) 코딩 탭이 만든 생성 파일도 삭제
rm -rf /path/to/Omni-node/workspace/coding/runs/*
```

## 5) 실동작 회귀 테스트 가이드 (수동)

## 5-1) 사전 조건

- 서버 실행 상태
- 웹 대시보드 접속 및 OTP 인증 완료
- 텔레그램 봇 연동 완료

## 5-2) 대화/코딩 분리 회귀

1. 대화탭(`chat:single`)에 문장 입력  
   예: `내 닉네임은 하빈이야`
2. 코딩탭(`coding:single`)에서 질문  
   예: `내 닉네임이 뭐였지?`
3. 기대 결과  
- 코딩탭에서 대화탭 문맥을 그대로 답하지 않아야 함(스코프 분리)

## 5-3) 대화탭 메모리 초기화 회귀

1. 대화탭에서 메모리성 문장 1~2개 입력
2. 대화 목록 패널 `메모리 초기화` 클릭 (대화탭 상태에서)
3. 같은 질문 재질의  
   예: `내 닉네임이 뭐야?`
4. 기대 결과  
- 이전 대화 기억을 참조하지 않아야 함
- 대화 리스트/메모리 노트 목록이 갱신되어야 함

## 5-4) 코딩탭 메모리 초기화 회귀

1. 코딩탭에서 메모리성 문장 입력  
   예: `우리 프로젝트 코드명은 X123이야`
2. 코딩탭에서 `메모리 초기화` 클릭
3. 동일 질문 재질의  
   예: `프로젝트 코드명이 뭐였지?`
4. 기대 결과  
- 초기화 후에는 이전 코딩탭 문맥 기억이 사라져야 함

## 5-5) 텔레그램 `/memory clear` 회귀

1. 텔레그램에서 일반 대화 1~2회 진행
2. 텔레그램에서 `/memory clear` 실행
3. 직전 문맥 질문  
   예: `방금 내가 뭐라고 했지?`
4. 기대 결과  
- 텔레그램에서 `ok: scope=telegram ...` 형태 성공 응답
- 직전 문맥을 유지하지 않아야 함
- 웹 대화탭(chat:single)에서도 동일 문맥이 비워진 상태여야 함

## 5-6) 대화탭 ↔ 텔레그램 연동 회귀

1. 텔레그램에서 `내 별명은 오미야` 입력
2. 웹 대화탭(chat:single)에서 관련 질문  
   예: `내 별명이 뭐야?`
3. 기대 결과  
- 연동이 정상이라면 대화탭에서 텔레그램 문맥을 참조 가능

## 6) 운영 권장

- 일반 운영: 개별 대화 삭제 + 필요 시 스코프별 `메모리 초기화`
- 강제 리셋: 4번 수동 전체 초기화
- 초기화 전 백업:

```bash
mkdir -p ~/.omninode-backup
cp -R ~/.omninode ~/.omninode-backup/omninode-$(date +%Y%m%d-%H%M%S)
```
