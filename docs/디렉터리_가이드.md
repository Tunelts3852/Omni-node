# Omni-node 디렉터리 가이드

업데이트 기준: 2026-03-08

이 문서는 루트부터 하위 모듈까지 “어디에 무엇이 들어 있는지”를 빠르게 찾기 위한 색인입니다.

현재 canonical 레이아웃은 `apps/`, `docs/`, `workspace/`입니다. 루트의 기존 `omninode-*`, `coding`, `runtime`, 문서 파일은 하위 호환을 위한 심볼릭 링크입니다.

## 3. 작업공간과 산출물

### 3.1 `workspace/coding/`

이 디렉터리는 제품 소스가 아니라 작업 대상 워크스페이스입니다.

주요 하위 경로:

| 경로 | 설명 |
|---|---|
| `hello_world/` | 간단한 HTML/Python 예제 |
| `recreated/` | 재생성 예제 |
| `task/` | 작업용 파일 |
| `tetris/` | 테트리스 예제 |
| `_routine_prompts/` | 루틴 생성용 프롬프트 |
| `routines/` | 루틴 실행 산출물 |
| `venv/` | 저장소 안에 들어 있는 Python 가상환경 |

즉, `workspace/coding/`은 앱 구현이 아니라 앱이 다루는 작업 공간입니다.

### 3.2 `workspace/.runtime/`

guard 회귀 실험 결과가 쌓이는 곳입니다.

예:

- `guard_retry_timeline.json`
- `gateway_health.json`
- `memory-index/main.sqlite*`
- 각 loop별 bundle JSON

### 3.3 `workspace/runtime/`

현재 상태 스냅샷용 소규모 디렉터리입니다.

### 3.4 위생 관점에서 보는 분류

| 경로 | 분류 | 청소 판단 |
|---|---|---|
| `workspace/coding/venv/` | 재생성 가능한 캐시 | 필요 없으면 삭제 가능 |
| `node_modules/` | 재생성 가능한 캐시 | `npm ci`로 복구 가능 |
| `apps/omninode-middleware/bin/`, `apps/omninode-middleware/obj/` | 빌드 산출물 | 삭제 가능 |
| `workspace/.runtime/` | 회귀 결과/임시 분석물 | 보관 이유가 없으면 정리 가능 |
| `workspace/runtime/` | 현재 세션 상태 스냅샷 | 세션 종료 후 필요 없으면 정리 가능 |
| `workspace/coding/routines/` | 실행 결과 이력 | 과거 기록이 필요 없을 때만 정리 |
| `~/.omninode/*` | 영속 상태 원본 | 삭제 전에 보존 여부를 먼저 판단 |

기본 예시 경로는 항상 `workspace/coding` 기준으로 기억하고, 루트 `coding`은 하위 호환 alias로만 봅니다.

## 7. 저장소를 볼 때의 해석 기준

이 저장소는 아래 네 층을 동시에 담고 있습니다.

1. 제품 런타임 소스
2. 운영 문서와 계획서
3. 회귀 테스트와 아티팩트
4. 실험/작업용 워크스페이스

따라서 파일 수가 많다고 해서 모두 제품 코드인 것은 아닙니다. 실제 구현 중심은 `apps/omninode-core/`, `apps/omninode-middleware/`, `apps/omninode-dashboard/`, `apps/omninode-sandbox/` 네 군데입니다.
