# Omni-node 환경변수 및 상태 파일

업데이트 기준: 2026-03-07

전체 목록은 `omninode-middleware/src/AppConfig.cs`가 가장 정확합니다. 이 문서는 실제 운영과 재실행에 자주 쓰는 항목만 추려 정리합니다.

## 1. 시크릿 로딩 우선순위

LLM/텔레그램 시크릿은 아래 순서로 로드됩니다.

1. macOS Keychain
2. 로컬 secure store
3. `*_FILE`
4. 직접 환경변수

중요 사항:

- 직접 환경변수는 deprecated 경고를 남깁니다.
- `*_FILE`은 권한이 `0600` 또는 그보다 엄격해야 합니다.
- 로컬 secure store 기본 경로는 `~/.config/omninode/secrets.json`입니다.

## 2. 자주 쓰는 환경변수

### 2.1 서버/경로

| 환경변수 | 기본값 | 의미 |
|---|---|---|
| `OMNINODE_WS_PORT` | `8080` | HTTP/WebSocket 포트 |
| `OMNINODE_CORE_SOCKET_PATH` | `/tmp/omninode_core.<uid>.sock` | 코어 UDS 소켓 경로 |
| `OMNINODE_DASHBOARD_INDEX` | 자동 탐색 | 대시보드 `index.html` 경로 |
| `OMNINODE_WORKSPACE_ROOT` | 자동 탐색 | 코딩 작업공간 루트 |
| `OMNINODE_SANDBOX_EXECUTOR` | `../omninode-sandbox/executor.py` | Python 샌드박스 실행기 경로 |
| `OMNINODE_GATEWAY_STARTUP_PROBE` | `false` | 게이트웨이 부트 프로브 활성화 |
| `OMNINODE_ENABLE_LOCAL_OTP_FALLBACK` | `true` | 텔레그램 실패 시 로컬 OTP fallback 사용 |

### 2.2 제공자/모델

| 환경변수 | 기본값 | 의미 |
|---|---|---|
| `OMNINODE_GROQ_MODEL` | `meta-llama/llama-4-scout-17b-16e-instruct` | 기본 Groq 모델 |
| `OMNINODE_GEMINI_MODEL` | `gemini-3-flash-preview` | 일반 Gemini 모델 |
| `OMNINODE_GEMINI_SEARCH_MODEL` | `gemini-3.1-flash-lite-preview` | 검색 전용 Gemini 모델 |
| `OMNINODE_CEREBRAS_MODEL` | `gpt-oss-120b` | 기본 Cerebras 모델 |
| `OMNINODE_COPILOT_MODEL` | `gpt-5-mini` | 기본 Copilot 모델 |
| `OMNINODE_CODEX_MODEL` | `gpt-5.4` | 기본 Codex 모델 |
| `OMNINODE_COPILOT_BIN` | `gh` | GitHub CLI 경로 |
| `OMNINODE_COPILOT_DIRECT_BIN` | `copilot` | Copilot 직접 바이너리 |
| `OMNINODE_CODEX_BIN` | `codex` | Codex CLI 바이너리 |

### 2.3 시크릿 입력

| 항목 | 직접 환경변수 | 파일 환경변수 |
|---|---|---|
| Telegram Bot Token | `OMNINODE_TELEGRAM_BOT_TOKEN` | `OMNINODE_TELEGRAM_BOT_TOKEN_FILE` |
| Telegram Chat ID | `OMNINODE_TELEGRAM_CHAT_ID` | `OMNINODE_TELEGRAM_CHAT_ID_FILE` |
| Groq API Key | `OMNINODE_GROQ_API_KEY` | `OMNINODE_GROQ_API_KEY_FILE` |
| Gemini API Key | `OMNINODE_GEMINI_API_KEY` | `OMNINODE_GEMINI_API_KEY_FILE` |
| Cerebras API Key | `OMNINODE_CEREBRAS_API_KEY` | `OMNINODE_CEREBRAS_API_KEY_FILE` |
| Codex API Key | `OMNINODE_CODEX_API_KEY` | `OMNINODE_CODEX_API_KEY_FILE` |

추가 제한:

- `OMNINODE_TELEGRAM_ALLOWED_USER_ID`: 허용할 텔레그램 사용자 제한

### 2.4 실행 한도/타임아웃

| 환경변수 | 기본값 | 의미 |
|---|---|---|
| `OMNINODE_CHAT_MAX_OUTPUT_TOKENS` | `8192` | 대화 최대 출력 토큰 |
| `OMNINODE_CODING_MAX_OUTPUT_TOKENS` | `16384` | 코딩 최대 출력 토큰 |
| `OMNINODE_LLM_TIMEOUT_SEC` | `20` | 일반 LLM 요청 타임아웃 |
| `OMNINODE_CODE_EXEC_TIMEOUT_SEC` | `120` | 코드 실행 타임아웃 |
| `OMNINODE_GEMINI_WEB_TIMEOUT_MS` | `30000` | Gemini grounding 웹 타임아웃 |
| `OMNINODE_WS_COMMANDS_PER_MINUTE` | `30` | WebSocket 분당 명령 제한 |
| `OMNINODE_WS_MAX_MESSAGE_BYTES` | `157286400` | WebSocket 최대 메시지 바이트 |

## 3. 기본 상태 파일 위치

상태 루트는 기본적으로 `~/.omninode`입니다. 홈 디렉터리를 얻지 못하면 `/tmp`로 fallback 합니다.

### 3.1 `~/.omninode` 아래 주요 파일

| 경로 | 의미 |
|---|---|
| `llm_usage.json` | LLM 사용량 상태 |
| `copilot_usage.json` | Copilot 사용량 상태 |
| `conversations.json` | 대화 저장소 |
| `auth_sessions.json` | 인증 세션 저장소 |
| `guard_retry_timeline.json` | guard retry timeline 상태 |
| `gateway_health.json` | gateway health 상태 |
| `gateway_startup_probe.json` | startup probe 상태 |
| `routines.json` | 루틴 상태 저장소 |
| `memory-notes/` | 메모리 노트 디렉터리 |
| `code-runs/` | 코드 실행 결과 디렉터리 |
| `memory-index/main.sqlite` | 메모리 검색 인덱스 DB |

### 3.2 `/tmp` 아래 주요 파일

코어와 일부 기본 경로는 `/tmp`를 직접 사용합니다.

| 경로 패턴 | 의미 |
|---|---|
| `/tmp/omninode_core.<uid>.sock` | 코어 UDS 소켓 |
| `/tmp/omninode.<uid>.lock` | 코어 단일 인스턴스 락 |
| `/tmp/omninode_audit.log` | 감사 로그 기본 위치 |

## 4. 운영에 자주 필요한 조합

### 4.1 최소 웹 대시보드 실행

```bash
export OMNINODE_WORKSPACE_ROOT="/Users/songhabin/Omni-node/coding"
export OMNINODE_ENABLE_LOCAL_OTP_FALLBACK=1
```

### 4.2 검색 기능 포함 실행

```bash
export OMNINODE_GEMINI_API_KEY_FILE="$HOME/.omni/keys/gemini_api_key"
export OMNINODE_GEMINI_SEARCH_MODEL="gemini-3.1-flash-lite-preview"
```

### 4.3 텔레그램 포함 실행

```bash
export OMNINODE_TELEGRAM_BOT_TOKEN_FILE="$HOME/.omni/keys/telegram_bot_token"
export OMNINODE_TELEGRAM_CHAT_ID_FILE="$HOME/.omni/keys/telegram_chat_id"
export OMNINODE_TELEGRAM_ALLOWED_USER_ID="123456789"
```

## 5. 알아두면 좋은 구현 포인트

- 대시보드 경로는 실행 경로 기준으로 자동 탐색합니다.
- 워크스페이스 루트도 자동 탐색하지만, 실제 운영에서는 `OMNINODE_WORKSPACE_ROOT`를 명시하는 편이 안전합니다.
- Codex는 API Key가 없더라도 CLI 로그인 상태가 살아 있으면 사용 가능할 수 있습니다.
- 시크릿 secure file은 권한이 느슨하면 읽지 않습니다.
