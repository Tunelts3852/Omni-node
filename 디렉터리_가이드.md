## 3. 작업공간과 산출물

### 3.1 `coding/`

이 디렉터리는 제품 소스가 아니라 작업 대상 워크스페이스입니다.

주요 하위 경로:

| 경로 | 설명 |
|---|---|
| `hello_world/` | 간단한 HTML/Python 예제 |
| `recreated/` | 재생성 예제 |
| `task/` | 작업용 파일 |
| `tetris/` | 테트리스 예제 |
| `_routine_prompts/` | 루틴 생성용 프롬프트 |
| `routines/` | 루틴 실행 산출물 |
| `venv/` | 저장소 안에 들어 있는 Python 가상환경 |

즉, `coding/`은 앱 구현이 아니라 앱이 다루는 작업 공간입니다.

### 3.2 `.runtime/`

guard 회귀 실험 결과가 쌓이는 곳입니다.

예:

- `guard_retry_timeline.json`
- `gateway_health.json`
- `memory-index/main.sqlite*`
- 각 loop별 bundle JSON

### 3.3 `runtime/`

현재 상태 스냅샷용 소규모 디렉터리입니다.

## 7. 저장소를 볼 때의 해석 기준

이 저장소는 아래 네 층을 동시에 담고 있습니다.

1. 제품 런타임 소스
2. 운영 문서와 계획서
3. 회귀 테스트와 아티팩트
4. 실험/작업용 워크스페이스

따라서 파일 수가 많다고 해서 모두 제품 코드인 것은 아닙니다. 실제 구현 중심은 `omninode-core/`, `omninode-middleware/`, `omninode-dashboard/`, `omninode-sandbox/` 네 군데입니다.
